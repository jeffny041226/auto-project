# 快速开始指南

## 环境要求

- Python 3.11+
- Node.js 18+
- Docker & Docker Compose
- MiniMax API Key (用于 LLM)

---

## 第一步：克隆项目

```bash
cd /Users/jeffny/sources/4.project/claude_demo1
```

---

## 第二步：启动基础设施服务

```bash
cd /Users/jeffny/sources/4.project/claude_demo1/auto-project

# 启动 MySQL, Redis, MinIO, Nginx
docker-compose -f docker/docker-compose.yml up -d

# 查看服务状态
docker-compose -f docker/docker-compose.yml ps
```

预期输出：
```
NAME                    STATUS
auto-test-mysql         Up
auto-test-redis         Up
auto-test-minio         Up
auto-test-nginx         Up
```

---

## 第三步：配置 API Key

```bash
cd backend

# 复制环境变量模板
cp .env.example .env

# 编辑 .env 文件，填入你的 MiniMax API Key
vim .env
```

修改以下内容：
```bash
MINIMAX_API_KEY=你的MiniMax API Key
MINIMAX_GROUP_ID=你的Group ID
```

---

## 第四步：安装后端依赖

```bash
cd backend
pip install -r requirements.txt
```

---

## 第五步：启动后端服务

```bash
# 在 backend 目录下执行
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

后端启动成功后会看到：
```
INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     Application startup complete.
```

---

## 第六步：安装前端依赖并启动

新开一个终端：

```bash
cd frontend
npm install
npm run dev
```

前端启动成功后会看到：
```
VITE v5.x.x  ready in 500 ms

➜  Local:   http://localhost:3000/
```

## 第七步：访问 Web UI

打开浏览器访问：**http://localhost:3000**

### 7.1 注册账号

点击 "Register" 链接，输入用户名和密码进行注册。

## 常用命令

### 查看日志

```bash
# 后端日志
docker-compose -f docker/docker-compose.yml logs backend

# MySQL 日志
docker-compose -f docker/docker-compose.yml logs mysql
```

### 停止服务

```bash
# 停止所有服务
docker-compose -f docker/docker-compose.yml down

# 停止并删除数据卷
docker-compose -f docker/docker-compose.yml down -v
```

### 重启服务

```bash
docker-compose -f docker/docker-compose.yml restart
```

## 常见问题

### 1. 端口被占用

如果 3000 或 8000 端口被占用，可以修改配置：

```bash
# 后端端口修改 (在 uvicorn 命令中指定)
uvicorn app.main:app --reload --host 0.0.0.0 --port 8001

# 前端端口修改 (在 vite.config.ts 中修改)
```

### 2. 数据库连接失败

```bash
# 检查 MySQL 是否正常运行
docker-compose -f docker/docker-compose.yml ps mysql

# 尝试重启 MySQL
docker-compose -f docker/docker-compose.yml restart mysql
```

### 3. MiniMax API 调用失败

```bash
# 检查 .env 配置
cat backend/.env | grep MINIMAX

# 验证 API Key
curl -H "Authorization: Bearer $MINIMAX_API_KEY" \
  https://api.minimax.chat/v
```

### 4. 前端无法连接后端

检查浏览器控制台是否有 CORS 错误。确认 Nginx 配置正确：

```bash
cat docker/nginx/nginx.conf
```
