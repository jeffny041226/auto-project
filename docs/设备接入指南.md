## 接入方式

### 方式一：ADB 网络连接（推荐）

设备开启网络 ADB，平台直接通过 ADB 控制：

```bash
# 1. 设备端：开启网络 ADB
adb tcpip 5555

# 2. 平台端：连接设备
POST /api/v1/devices/connect?device_ip=192.168.1.100&device_port=5555
```

**优点**：简单，无需额外软件
**缺点**：需要设备开启开发者模式和网络 ADB

### 方式二：设备 Agent

在设备上运行 Python Agent，通过 WebSocket 与平台通信：

```bash
# 设备端
python agent/device_agent.py \
  --device-id my-phone-001 \
  --server http://your-server:8000
```

**优点**：支持防火墙后的设备
**缺点**：需要设备安装 Python 环境

### 方式三：Maestro CLI

使用 Maestro 自带的设备管理模式：

```bash
# 平台启动 Maestro Server
maestro server --port 7000

# 设备通过 Maestro CLI 连接
maestro device add --name "My Phone" --udid <serial>
```

## 快速开始

### 步骤 1：准备 Android 设备

1. 开启 **开发者选项**
2. 开启 **USB 调试**
3. （可选）开启 **网络 ADB**：
   ```bash
   # 用 USB 连接电脑后执行
   adb tcpip 5555
   ```

### 步骤 2：注册设备到平台

**通过 API 注册：**
```bash
curl -X POST http://localhost:8000/api/v1/devices \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "device_id": "samsung-s21-001",
    "device_name": "Samsung Galaxy S21",
    "os_version": "Android 13",
    "model": "SM-G991B"
  }'
```

**通过前端：**
访问 `设备管理` 页面，点击 `添加设备`。

### 步骤 3：连接设备

**USB 设备自动发现：**
```bash
# 查看已连接的设备
GET /api/v1/devices/discover
```

**网络设备手动连接：**
```bash
POST /api/v1/devices/connect?device_ip=192.168.1.100
```

### 步骤 4：验证连接

```bash
# 获取设备详情
GET /api/v1/devices/samsung-s21-001/info
```

响应示例：
```json
{
  "device": {...},
  "live_info": {
    "serial": "samsung-s21-001",
    "model": "Samsung Galaxy S21",
    "android_version": "13",
    "resolution": {"width": 1080, "height": 2400}
  },
  "current_app": "com.tencent.mm"
}
```

## 使用 Python Agent

### 安装依赖

```bash
# 在设备上安装 (需要 Python 3.8+)
pip install websockets

# 或使用 Termux
pkg install python
pip install websockets
```

### 运行 Agent

```bash
python agent/device_agent.py \
  --device-id my-phone-001 \
  --server http://your-server:8000 \
  --api-key optional_api_key
```

### Agent 支持的命令

| Action | 说明 | 参数 |
|--------|------|------|
| `shell` | 执行 Shell 命令 | `command` |
| `tap` | 点击坐标 | `x`, `y` |
| `swipe` | 滑动 | `start_x`, `start_y`, `end_x`, `end_y`, `duration` |
| `input_text` | 输入文本 | `text` |
| `screenshot` | 截图 | - |
| `start_app` | 启动应用 | `package`, `activity` |
| `press_key` | 按键 | `keycode` (4=返回, 3=主页) |

## ADB 命令参考

### 常用 ADB 命令

```bash
# 连接设备
adb connect 192.168.1.100:5555

# 断开连接
adb disconnect 192.168.1.100:5555

# 查看已连接设备
adb devices -l

# 获取屏幕分辨率
adb shell wm size

# 获取 Android 版本
adb shell getprop ro.build.version.release

# 获取设备型号
adb shell getprop ro.product.model

# 安装 APK
adb install app.apk

# 卸载应用
adb uninstall com.example.app

# 截图
adb shell screencap -p /sdcard/screen.png
adb pull /sdcard/screen.png

# 录屏
adb shell screenrecord /sdcard/video.mp4
adb pull /sdcard/video.mp4

# 点击
adb shell input tap 500 1000

# 滑动
adb shell input swipe 500 1500 500 500 500

# 输入文本
adb shell input text "Hello"

# 按键
adb shell input keyevent 4   # 返回
adb shell input keyevent 3    # 主页
adb shell input keyevent 82   # 菜单
```

### 按键码参考

| 按键 | KeyCode |
|------|---------|
| 返回 | 4 |
| 主页 | 3 |
| 菜单 | 82 |
| 电源 | 26 |
| 音量+ | 24 |
| 音量- | 25 |

## 故障排查

### 设备连接失败

1. **检查网络**
   ```bash
   ping 192.168.1.100
   ```

2. **检查 ADB 端口**
   ```bash
   telnet 192.168.1.100 5555
   ```

3. **重启 ADB 服务**
   ```bash
   adb kill-server
   adb start-server
   ```

### 设备离线

1. 检查设备是否断开连接
2. 重新连接：
   ```bash
   adb connect 192.168.1.100:5555
   ```
3. 更新平台设备状态

### Agent 连接失败

1. 确认平台 WebSocket 可达
2. 检查防火墙设置
3. 确认设备 ID 唯一

## 网络要求

| 组件 | 端口 | 说明 |
|------|------|------|
| 平台 API | 8000 | REST API |
| 平台 WebSocket | 8000 | 设备连接 |
| ADB | 5555 | 设备控制 |

**防火墙建议：**
- 平台服务器开放 8000 端口
- 设备到服务器： outbound 8000, 5555
- 服务器到设备： inbound 8000

## 最佳实践

1. **使用稳定的网络连接**
   - 推荐使用有线网络或近距离 WiFi
   - 避免高延迟网络

2. **设备保持唤醒**
   ```bash
   # 禁用屏幕超时
   adb shell settings put system screen_off_timeout 2147483647
   ```

3. **关闭设备锁屏密码**（测试环境）
   ```bash
   adb shell settings put secure lock_screen_lock_after_timeout 0
   ```

4. **预留设备资源**
   - 关闭后台应用
   - 保持足够电量或接电源

5. **定期检查设备状态**
   - 使用心跳机制
   - 自动重连
