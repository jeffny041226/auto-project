生成脚本：
1.agent分析截图进行操作对于点击的操作，根据点击的坐标位置结合uiautomator dump 的 XML解析出对应的元素信息，例如：
文字: 微信, id: com.tencent.mm:id/icon, 坐标: [100,200,200,300]
2.将解析出的元素信息和ai的思考过程，在执行完成后通过ws发送所有的步骤给后端服务，数据类型为JSON如下
{
    "steps": [
        {
        "action": "click",
        "element_info": {
            "text": "微信",
            "id": "com.tencent.mm:id/icon",
            "coordinates": [100, 200, 200, 300]
        },
        "ai_thought_process": "点击微信图标，准备进入微信应用"
        },
         {
        "action": "click",
        "element_info": {
            "text": "手机号登陆",
            "id": "com.tencent.mm:id/phone_login",
            "coordinates": [100, 200, 200, 300]
        },
        "ai_thought_process": "点击手机号，准备进入手机号登陆界面"
        },
        {
        "action": "back",
        "element_info": {
            "text": "",
            "id": "",
            "coordinates": null
        },
        "ai_thought_process": "显然这不是我想要进入的页面，让我点击返回键，准备返回上一界面"
        }，
                 {
        "action": "click",
        "element_info": {
            "text": "账号登陆",
            "id": "com.tencent.mm:id/login",
            "coordinates": [100, 200, 200, 300]
        },
        "ai_thought_process": "点击账号登陆，准备进入账号登陆界面"
        },
    ]
}。后端服务的llm可以根据这些信息进行提纯，只保留有用的信息，即只保留第一步和第四部分，生成最终的操作步骤如下：
{
    "steps": [
        {
        "action": "click",
        "element_info": {
            "text": "微信",
            "id": "com.tencent.mm:id/icon",
            "coordinates": [100, 200, 200, 300]
        },
        "ai_thought_process": "点击微信图标，准备进入微信应用"
        },
         {
        "action": "click",
        "element_info": {
            "text": "账号登陆",
            "id": "com.tencent.mm:id/login",
            "coordinates": [100, 200, 200, 300]
        },
        "ai_thought_process": "点击账号登陆，准备进入账号登陆界面"
        },
    ]
}
后端的llm可以根据这些步骤生成可执行的maestro脚本。
注意关于步骤提纯和脚本生成的prompt要写的尽量详尽
